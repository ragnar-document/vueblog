    # NO.3 HTML表单元素

    表单是用来收集用户输入的内容

    form是定义html表单的标签

    input是表单最重要的标签元素 拥有多种type，常用属性有checked，placeholder，text,password,button,submit,value,radio，reset重置

    Select,option下拉列表标签

    Textarea 标签定义一个多行的文本输入控件cols规定文本可见宽度

    label定义了input标签的标题

    fieldest定义了一组相关的表单元素，并使用外框包含起来

    legend定义了fieldest元素标题

    <datalist> 标签规定了 <input> 元素可能的选项列表。

<datalist> 标签被用来在为 <input> 元素提供"自动完成"的特性。用户能看到一个下拉列表，里边的选项是预先定义好的，将作为用户的输入数据。

请使用 <input> 元素的 list 属性来绑定 <datalist> 元素。

    以下是使用Bootstrap进行表单验证的方法：

    • HTML表单验证通过CSS的两个伪类应用，:invalid以及:valid。它适用于<input>，<select>和<textarea>元素。
    • Bootstrap将:invalid和类的范围限定:valid为父.was-validated类，通常应用于<form>。否则，没有值的任何必填字段在页面加载时显示为无效。这样，您可以选择何时激活它们（通常在尝试提交表单之后）。
    • 作为备用，.is-invalid并且.is-valid类别可以用来代替伪类用于服务器端验证。他们不需要.was-validated父类。
    • 由于CSS的工作方式存在限制，我们无法（目前）<label>在没有自定义JavaScript帮助的情况下将样式应用于DOM中的表单控件之前。
    • 所有现代浏览器都支持约束验证API，这是一系列用于验证表单控件的JavaScript方法。
    • 反馈消息可以使用浏览器默认值（每个浏览器不同，通过CSS不可用）或我们的自定义反馈样式以及其他HTML和CSS。
    • 您可以使用setCustomValidityJavaScript 提供自定义有效性消息。

    考虑到这一点，请考虑我们的自定义表单验证样式，可选服务器端类和浏览器默认值的以下演示。

<script>
// Example starter JavaScript for disabling form submissions if there are invalid fields
(function() {
  'use strict';
  window.addEventListener('load', function() {
    // Fetch all the forms we want to apply custom Bootstrap validation styles to
    var forms = document.getElementsByClassName('needs-validation');
    // Loop over them and prevent submission
    var validation = Array.prototype.filter.call(forms, function(form) {
      form.addEventListener('submit', function(event) {
        if (form.checkValidity() === false) {
          event.preventDefault();
          event.stopPropagation();
        }
        form.classList.add('was-validated');
      }, false);
    });
  }, false);
})();
</script>
