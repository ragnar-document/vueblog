# vue边学边自问

# 1.什么是vue

vue.js是一个开源的，用于构建用户界面的渐进式的JavaScript框架，vue被设计为可以自底向上逐层应用，vue的核心库只关心视图层

# 2.对比其他框架

# react 和 vue

相同处：

使用virtual DOM（虚拟dom）

提供响应式和组件化的视图组件

将注意力集中保持在核心库，而将其他功能入路由和全局状态管理交给相关的库

性能

在react中组件状态变化会以该组件为根重新渲染整个组件数；vue中组件的依赖是在渲染过程中自动追踪的，系统会单独渲染该组件

html与css

在react中一切皆可JavaScript，而vue更趋向于经典的web技术对于初学者来说vue更容易学习

# angular 和 vue

复杂性：在API和设计两方面vue更简单

灵活性：你必须遵守angular制定的规则，而vue更侧重让你决定

数据绑定：angular使用的是双向绑定，vue在不同组件中强制使用单向数据流，这让应用中的数据更加清晰

组件与指令：vue中指令只封装dom操作 在angular中每件事都由指令来做，而组件只是一种特殊指令

性能：Vue 有更好的性能，并且非常非常容易优化，因为它不使用脏检查。

# 3.vue的主要特点

虚拟DOM，组件，模版，路由，重量轻

# 4.vue的生命周期方法是什么

生命周期是一个窗口，可以了解你正在使用的库如何在幕后工作，通过这些钩子，你将了解何时创建组件添加到Dom，更新或销毁组件

beforeCreate(数据创建前) >> created(创建) >> beforeMount(即将挂载) >> mounted(载入) >> beforeUpdata(数据更新前) >> updated(更新) >> activated(激活) >> deactiveated(停用) >> beforeDestroy(实例销毁前调用) >> destroyed(销毁) >> errorCaptured(捕获子孙组件错误)

# 5.什么是条件指令

vue中提供了一组指令，用于根据条件显示或隐藏元素

可用的指令有 v-if ，v-else ，v-else-if 和v-show，v-for

v-if用于条件性的渲染内容，只有当条件为true值时才会渲染

<h1 v-if="awesome">Vue is awesome!</h1>
1

也可以加上v-else

<h1 v-else>Oh no 😢</h1>
1

v-else-if，顾名思义，充当 v-if 的“else-if 块”，可以连续使用

v-show 不支持templa元素，v-show只是操作css切换实际上时存在dom的而v-if是惰性的当条件为真的时候才会生成dom。在开销大的情况下v-show会更好一点

v-if和v-for不推荐一起使用，并且v-for优先级更高

当想渲染部分节点时这种优先级还是非常有用的例如todo

# 11.数组的变异方法与非变异方法

移步官网

简单来说就是更改原数组的方法就叫变异方法，调用原来数组形成新的数组叫做非变异方法，因为它并没有改变原有的数组。

注意事项：由于 JavaScript 的限制，Vue 不能检测以下数组的变动

  1. 当你利用索引直接设置一个数组项时，例如：vm.items[indexOfItem] = newValue
  2. 当你修改数组的长度时，例如：vm.items.length = newLength

解决的方法

解决第一类

//Vue.set
Vue.set(vm.items,indexOfItem,newValue)

//Array.prototype.splice
vm.items.splice(indexOfItem,1,newValue)

//你也可以使用 vm.$set 实例方法，该方法是全局方法 Vue.set 的一个别名
vm.$set(vm.items,indexOfItem,newValue)

解决第二类

vm.items.splice(newLength)

# 14.有哪些事件修饰符和按键修饰符，它们的作用是什么

常用的事件修饰符有.stop/.prevent/.once/.self/.passove

注意点是修饰符可以串联，但是顺序不能随便否则会出问题

例如：v-on:click.prevent.self 会阻止所有点击

stop：阻止事件向上冒泡，简单来说父级如果有点击事件的话不使用stop阻止那么儿子跑出去的话父亲也会一起跑。

prevent：提交事件时不重新加载页面

once：事件只执行一次如果已经使用就会失效

self：只当在 event.target 是当前元素自身时触发处理函数

passove：滚动事件的默认行为 (即滚动行为) 将会立即触发，可以提高移动端的性能

常见的按键修饰符有.enter/.tab/.delete/.esc/.up/.down/.left/.right

还可以自定义修饰符

// 可以使用 v-on:keyup.f1 Vue.config.keyCodes.f1 = 112

系统修饰符有.ctrl/.alt/.shift/.meta

v-model中的修饰符.lazy/.number/.trim

lazy:惰性更新添加该修饰符可以于change事件同步，而不是实时同步

number：将用户输入的值转化为数值类型

trim：去掉首尾空白符

仅在按下相应按键才触发鼠标或键盘的监听器

<!-- Ctrl + Click -->
<div @click.ctrl="doSomething">Do something</div>

# 15.组件间的通信

通过prop向子组件传递数据，prop时你可以在组件上组册的一些自定义属性，当一个值传递个prop特性时它就成为那个组件实例的一个属性我们可以用一个 props 选项将其包含在该组件可接受的 prop 列表中，一个组件可以拥有任意数量的prop，在实例中访问这个值就像访问data一般

子传父可以使用$emit触发当前实例事件附加参数会都会传给监听器回调
