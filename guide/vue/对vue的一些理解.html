# 对vue的一些理解

​ vue是华裔开发者尤雨溪开发的一款mvvm渐进式JavaScript框架。在文档视频中介绍了vue如何让我们更专注于视图层以及vue实现的一些方便的交互。

​ vue的基本原理我看了一下文档是这样介绍的在组件渲染时创建虚拟dom树通过事件触发调用data中的getting和setting方法，使用watch来动态渲染组件内容。并且vue改变dom是一种异步队列操作，并缓冲在同一事件循环中发生的所有数据变更。多次触发只会实现第一次通过标记避免重复数据带来的麻烦。

(使用 Object.defineProperty 把这些属性全部转为 getter/setter)

​ 在实际应用中的确体验到了该核心带来的便利，在使用数据渲染列表的时候感觉到异常的轻松。

关于路由的一些理解，在页面中还有离不开的就是页面路由，vue中把路由分到vue-router的核心库中，我们可以把页面看出是一个个组件通过将组件映射到路由上然后再告诉Vue-Router再哪里渲染，在vue的路由中我们还能使用编程的手法去写路由所以会感到非常的便捷和开发，在页面复杂且多的时候命名路由的好处就体现了。它能使我们更好的管理每一条路由更加的清晰可控。路由守卫在后台中会经常使用到例如管理人员登录，以及对用户查看的权限都有帮助。
